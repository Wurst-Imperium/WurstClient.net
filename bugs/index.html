  1. Search it in the bug tracker

    Before reporting a bug, search through the existing bug reports in our bug tracker to see if it has been reported already.

    {% comment %}

    Google only updates once per day, so people wouldn't be able to find the latest bug reports using this search bar. It's quite unfortunate, since Google's search is much more user-friendly than Github's one, but the primary purpose of this search bar is to prevent people from posting duplicates.

    {% endcomment %}

    The next steps depend on whether or not you found an existing bug report.

  2. Vote on the bug report

    If you found an existing bug report for the bug that you are experiencing, you can vote on the bug report to let us know that you are having the same problem.

    How to vote on a bug report:

  3. Help out where you can

    If you have more information about this bug than the bug report contains, share your knowledge by adding a comment. Maybe the author of the bug report failed to attach a crash log or you found a different strategy to reproduce the bug.

    If you are a coder and you know how to fix this bug, make a Pull Request on the Wurst Client repository.

  4. Read the bug tracker rules

    If you cannot find an existing bug report for the bug that you are experiencing, you can create a new bug report.

    But first, you have to read the bug tracker rules. Your bug report must comply with these rules, or else you will be banned!

  5. Create a new bug report

    OK, now you are ready to make a new bug report.

    You will be presented with a bug report template. Just fill it out with correct information, add a title to the bug report and click the "Submit" button.
